# Установка

## Требования

- Node.js 20+
- Git-репозиторий
- Claude Code (VS Code extension или CLI)

## Установка

```bash
npm install -g @engramm/dev-workflow
```

### Из исходников (для разработки)

```bash
cd dev-workflow
pnpm install
pnpm run build
npm link
```

## Инициализация в проекте

```bash
cd your-project
dev-workflow init
```

Команда `init` создаёт:

| Что | Где | Назначение |
|-----|-----|-----------|
| CLAUDE.md | `./CLAUDE.md` | Инструкции для Claude Code (команды, vault, workflow) |
| Vault | `.dev-vault/` | Хранилище знаний (stack, conventions, knowledge, gameplan, phases, plans) |
| Хуки | `.claude/settings.json` | 3 хука: SessionStart, SessionEnd, TaskCompleted |
| Permissions | `.claude/settings.json` | Разрешения: файлы, тесты, git, агенты, MCP |
| MCP-сервер | `.mcp.json` | Программный доступ к vault (26 tools) |
| Slash-команды | `.claude/skills/` | 35 skills: vault\_\_\* (9), session\_\_\* (3), git\_\_\* (4), workflow\_\_\* (7), плюс root (4 — `intake`, `profile`, `task`, `workflow`) и `obsidian-markdown` |
| StatusLine | `.claude/settings.json` | Статус проекта в нижней строке Claude Code |
| .gitignore | `.gitignore` | Записи по стеку (node\_modules, target/, \_\_pycache\_\_ и т.д.) |

### Permissions

`init` добавляет permissions в `.claude/settings.json`:

| Allow | Deny |
|-------|------|
| Read, Edit, Write | .env файлы |
| npm/pnpm test, run, build | git push |
| node, npx | git reset --hard |
| git status, diff, log, add, commit | rm -rf |
| Agent spawning, MCP tools | |

### MCP-сервер

MCP конфигурация в `.mcp.json` (Claude Code автоматически обнаруживает):

```json
{
  "mcpServers": {
    "dev-workflow": {
      "command": "node",
      "args": ["/path/to/dist/cli/index.js", "serve"]
    }
  }
}
```

Путь вычисляется автоматически при `init` — абсолютный, работает для npm link, глобальной установки и локальной зависимости.

### Engram (если используется)

Сокет ищется в порядке: `ENGRAM_SOCKET_PATH` env var → `./.engram/engram.sock` (per-project) → `~/.engram/engram.sock` (legacy fallback). Для миграции legacy memories запустите `engram migrate` из корня проекта.

## Проверка

```bash
dev-workflow doctor
```

Покажет состояние каждого компонента: vault, агенты, задачи, workflows, MCP config.

```bash
dev-workflow doctor --fix
```

Автоматическое исправление: перезапускает `init --force`.

## Веб-дашборд

После `init` локальный read-write дашборд поднимается одной командой:

```bash
dev-workflow web                  # http://127.0.0.1:3737
```

Один процесс Node, loopback-only (`127.0.0.1`, без `--host`-флага по дизайну). Шесть страниц — Overview / Vault / Tasks / Workflow / Engram / Settings. Multi-project навбар, live-обновления через SSE. Подробнее — [концепция «Дашборд»](/docs/concepts/dashboard).
